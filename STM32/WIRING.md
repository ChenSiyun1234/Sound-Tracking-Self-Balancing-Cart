# STM32 接线说明（一步一步）— 分立模块版自平衡小车

> 本版本用一块**通用 STM32F103C8T6「蓝色药丸 / Blue Pill」**，外加底盘自带的
> **TK-TB6612-MD220A 驱动板** + **MPU6050（GY-521）+ 江协 SSD1306 I²C OLED + HC-05 蓝牙 + HC-SR04 超声波**。
> 平衡环跑在 **MPU6050 数据就绪中断里（PA5 → EXTI9_5，下降沿，100Hz）**，姿态解算用商家板载的 **DMP**（四元数 → pitch/yaw）。
>
> ⚠️ 本文件以**真正编进固件的引脚**（`Driver/*.c`）为准。旧的 OpenCTR / RCT6 那套接线已作废，别再照那个接。
>
> 🔧 **编码器引脚已 remap 到耐 5V 脚**（PA15/PB3/PB4/PB5），所以 MD220A 输出的 5V 编码器信号**可直连，不再需要电平转换板**。

## 1. 供电架构（供电）— MD220A 是电源中心

**电池只接到 MD220A 一处**。MD220A 自带降压电路：输入 5.5–15V，板载 **RT8289 → 5V/5A**、**RT9013 → 3.3V/500mA**，并把 **5V / 3V3 / GND** 从它的「**单片机接口**」排引出来，去喂 STM32 和所有模块。所以你**不用再外加任何稳压**。

```
电池 (~6–7.4V, 2S 锂电)
   │
   └─► MD220A 电源输入 (J1：2P端子 / DC口，5.5–15V)
          │   ⚠️ 这个输入电压 = 电机 VM。MC130 额定 ~6V，别上 12V！（见 §注意5）
          │
          ├─► TB6612 ──► 电机 A / 电机 B          (VM = 输入电压)
          ├─► RT8289 降压 ──► 5V / 5A
          └─► RT9013 LDO  ──► 3.3V / 500mA
                 │
   MD220A「单片机接口」排把电源引出来 → 接 STM32 与各模块：
          ├─► 5V  ──► C8T6 的 "5V" 脚（板载 AMS1117 再生成 3.3V 核心）
          ├─► 5V  ──► HC-SR04 VCC、HC-05 VCC
          ├─► 3V3 ──► MPU6050 VCC、OLED VCC
          └─► GND ──► C8T6 GND + 每个模块 GND（单点星形地）

   编码器：由 MD220A 经「电机接口」在板上供电（5V），你不用单独给它供电；
           固件已把编码器引脚 remap 到耐 5V 的脚，所以 5V 信号可直连 STM32（见 §注意1）。
```

**轨的定义（后面表里说的"轨"都指这里）：**

| 轨 | 来自 | 挂在上面的东西 |
|---|---|---|
| **VM（电机轨）** | MD220A 输入电压（=你给 J1 的电压）| 只有 TB6612→电机；**=输入电压，用 ~6–7.4V** |
| **5V 轨** | MD220A `5V`（RT8289，5A）| C8T6 的 5V 脚、HC-SR04、HC-05 |
| **3.3V 轨** | MD220A `3V3`（RT9013，500mA）| MPU6050、OLED |
| **公共地** | MD220A `GND` ＝ 电池负极 | 所有 GND 必须汇到这一个点 |

**关键点：**
- **C8T6 用 MD220A 的 5V 喂 "5V" 脚**（板载 LDO 生成核心 3.3V）。**别往 `3V3` 脚同时灌电**。
- **3.3V 模块（MPU/OLED）直接吃 MD220A 的 3V3**（500mA 足够），不要去借蓝色药丸那颗小 LDO。
- **MC130 ~6V**：MD220A 的输入电压就是电机电压，固件满 PWM 会全开 →**输入用 ~6–7.4V，别上 12V**（见 §注意5）。
- **必须共地**（见 §注意6）。5V 轨靠近传感器处并一个 **100–470µF** 电容，吃电机电流抖动。

## 3. 一步一步接线（按模块）

### 第 1 步：MD220A「单片机接口」排 ↔ STM32（主排线，先接这个）

电机驱动、编码器信号、还有给 STM32/模块的电，**几乎全从 MD220A 这一排出来**。照下表一一对应：

| MD220A 单片机接口 | 接到 STM32 / 模块 | 作用 |
|---|---|---|
| **5V** | C8T6 **5V 脚**（同时分给 HC-SR04 / HC-05）| 供电 5V/5A |
| **3V3** | MPU6050 / OLED 的 3.3V | 供电 3.3V/500mA |
| **GND** | C8T6 GND + 所有模块 GND | 公共地（单点星形）|
| **PWMA** | **PA8** | 左电机调速（TIM1_CH1）|
| **AIN1** | **PB12** | 左电机方向 1 |
| **AIN2** | **PB13** | 左电机方向 2 |
| **PWMB** | **PA11** | 右电机调速（TIM1_CH4）|
| **BIN1** | **PB14** | 右电机方向 1 |
| **BIN2** | **PB15** | 右电机方向 2 |
| **STBY** | **PA4** | 高=使能，低=断力 |
| **E1A** | **PA15**（TIM2_CH1 remap，**5V 直连**）| 1 号编码器 A 相 |
| **E1B** | **PB3**（TIM2_CH2 remap，**5V 直连**）| 1 号编码器 B 相（与 E1A 同属 TIM2，成对）|
| **E2A** | **PB4**（TIM3_CH1 remap，**5V 直连**）| 2 号编码器 A 相 |
| **E2B** | **PB5**（TIM3_CH2 remap，**5V 直连**）| 2 号编码器 B 相（与 E2A 同属 TIM3，成对）|
| **ADC** | 不接 | 底盘电池分压采样口；本固件不读电池，留空 |

> 控制信号（PWMA/AINx/PWMB/BINx/STBY）是 STM32 的 3.3V 输出送进 TB6612（输入 ~2V 即为高），**直驱即可**。编码器那 4 根虽是 MD220A→STM32 的 5V 信号，但**固件已把它们 remap 到耐 5V 的脚（PA15/PB3/PB4/PB5），可直连，无需电平转换**（见 §注意1）。
> TB6612 真值表：`INx=1,0`→正转；`0,1`→反转；`0,0`→滑行；`1,1`→刹车；占空比定速度。

### 第 2 步：电机 + 电池 接到 MD220A

| MD220A | 接到 | 说明 |
|---|---|---|
| **电机接口（XH2.54 6P）×2** | 两个 MC130 电机的 6 芯线**直接插**（电机电源 + 霍尔编码器都在这根线里）| 别左右插反；编码器由 MD220A 在板上供电（5V）|
| **电源输入 J1（2P端子 / DC）** | 电池（~6–7.4V）| ⚠️ 此电压=电机 VM，MC130 额定 ~6V，**别上 12V**（见 §注意5）|

> 电机正反向、左右，**上电调试时再定**（见末尾清单）：装反了就翻该电机的 AIN1/AIN2（或 BIN1/BIN2）。

### 第 3 步：编码器（已 remap 到耐压脚，直连，无需电平转换）

编码器信号在 MD220A 上是 **5V**（板上霍尔供电写死 5V），但**固件把编码器定时器重映射到了耐 5V 的脚**，所以 E1A/E1B/E2A/E2B 从 MD220A「单片机接口」**直接连到 STM32 即可**（接线见第 1 步那张表）：

- 1 号编码器（TIM2）：**E1A → PA15，E1B → PB3**
- 2 号编码器（TIM3）：**E2A → PB4，E2B → PB5**

> **关键规则：同一个编码器的 A/B 两相必须落在同一个 TIM 上**——E1A+E1B 一起进 TIM2（PA15/PB3），E2A+E2B 一起进 TIM3（PB4/PB5），别拆开，否则读不出来。
> 哪个算「左/右」、A/B 谁先后，上电调试再定：某轮速度反馈反了就把那一对 A/B 对调；左右整体反了就把 E1↔E2 两对线对调。
> （PA15/PB3/PB4 是关 JTAG 后释放的脚，固件在关 JTAG 之后才初始化编码器，所以可用；SWD 仍保留。）

### 第 4 步：MPU6050（GY-521，⚠️ 3.3V）

| GY-521 脚 | 接到 | 说明 |
|---|---|---|
| **VCC** | **MD220A 3V3**（3.3V 轨）| 见 §注意(4)，让整条 I²C/INT 都在 3.3V |
| **GND** | **公共地** | |
| **SCL** | **PB10** | 软件 I²C 时钟（模块板载 ~4.7k 上拉到 VCC）|
| **SDA** | **PB11** | 软件 I²C 数据 |
| **INT** | **PA5** | 数据就绪中断 → EXTI9_5 下降沿，**100Hz 平衡环靠它触发** |
| **AD0** | 接地/悬空 | 地址 = 0x68 |
| **XDA / XCL** | 不接 | 辅助 I²C，本项目不用 |

> SCL/SDA 在 FT 的 PB10/PB11，但**整条 I²C+INT 必须 3.3V**——因为 INT 走**非 FT 的 PA5**。

### 第 5 步：OLED 屏（江协 SSD1306，I²C 4 脚）

固件把 OLED 电源也用 GPIO 顶出来了：**PB7 拉高当 VCC（~3.3V）、PB6 拉低当 GND**，软件 I²C 跑 **PB8(SCL)/PB9(SDA)**。

| OLED 脚 | 接到 STM32 | 说明 |
|---|---|---|
| **GND** | **PB6**（拉低）/ 或公共地 | 见 §注意(3) |
| **VCC** | **PB7**（拉高 ~3.3V）/ 或 MD220A 3V3 | 见 §注意(3) |
| **SCL** | **PB8** | 软件 I²C 时钟（开漏）|
| **SDA** | **PB9** | 软件 I²C 数据（开漏）|

> ⚠️ 用 PB7 当电源有限流隐患（见 §注意(3)）。**更稳：VCC 接 MD220A 3V3、GND 接公共地**，PB6/PB7 留空。必须是 **I²C 版 4 脚屏**。

### 第 6 步：HC-05 蓝牙（手机遥控用，UART）

UART **要交叉**：模块 TXD → 单片机 RX，单片机 TX → 模块 RXD。

| HC-05 脚 | 接到 | 说明 |
|---|---|---|
| **VCC** | **MD220A 5V**（5V 轨）| 模块板载稳压，数据线 3.3V；固件 USART2 = **9600**（HC-05 默认，无需 AT 改）|
| **GND** | **公共地** | |
| **TXD** | **PA3**（USART2_RX）| 模块 3.3V TX → 单片机 RX，正版 HC-05 是 3.3V，**可直连**；见 §注意 |
| **RXD** | **PA2**（USART2_TX）| 单片机 3.3V TX → 模块 RXD |
| **EN/KEY** | 不接（普通数据模式）| 只在进 AT 配置时才接 |
| **STATE** | 不接 | 连接状态指示输出，本项目不用 |

> ⚠️ PA3 是**非 FT 且零余量**。正版 HC-05 的 TXD 是 3.3V，安全；若换了某些杂牌 5V TX 模块，**PA3 会被打坏**——换模块前先量 TXD 静态电平 ~3.3V，是 5V 就要分压（串 1k + 对地 2k）。

**遥控指令（单字符，大小写都行；用 Serial Bluetooth Terminal 之类的蓝牙串口 App 发）：**

| 发送 | 作用 |
|---|---|
| `F` / `B` | 前进 / 后退（锁定，直到 `S` 或反向）|
| `L` / `R` | 左转 / 右转 |
| `S` | 停（vx=vw=0，仍保持平衡）|
| `O` | 开 / 关 超声波避障模式 |

> 配对默认 PIN `1234`/`0000`；App 里把 M1–M9 宏按钮各设成一个字母，就成了遥控面板。**没有"关平衡"指令**（那只会让车摔倒）——要停就发 `S`（只停行驶、继续平衡）或关电源开关。避障默认关，按 `O` 打开后，遇到前方 <25cm 会自动 停 → 原地扫描 → 转向最空方向 → 退出。

### 第 7 步：HC-SR04 超声波

| HC-SR04 脚 | 接到 | 说明 |
|---|---|---|
| **VCC** | **MD220A 5V**（5V 轨）| 模块要 5V 才能可靠发射 |
| **TRIG** | **PB0** | 单片机 3.3V 输出触发，HC-SR04 阈值 ~2.5V，3.3V 够了 |
| **ECHO** | **PA12** | ⚠️ ECHO 是 **5V** 脉冲，但 **PA12 是 FT 脚，可直连不分压**！见 §注意(2) |
| **GND** | **公共地** | |

> **别把 TRIG 和 ECHO 接反**：TRIG=PB0（输出，非 FT），ECHO=PA12（输入，FT）。反了的话 5V 的 ECHO 落到非 FT 的 PB0 上会出问题。

### 第 8 步：LED / 调试（可选）

| 信号 | STM32 引脚 | 说明 |
|---|---|---|
| 心跳 LED | **PC13** | 板载 LED，低有效，固件闪灯示活；弱后备域脚，别接负载/别灌 5V |
| 调试串口 TX | **PA9**（USART1_TX）→ USB-TTL 的 RX | 看 `printf` 日志 |
| 调试串口 RX | **PA10**（USART1_RX）→ USB-TTL 的 TX | PA10 是 FT，5V USB-TTL 也安全 |

## ⚠️ 注意（CAUTIONS — 上电前逐条核对）

1. **编码器已用耐 5V 脚，可直连——但别接错脚。** MD220A 给霍尔编码器供的是 **5V**（板上写死，改不了），所以 E1A/E1B/E2A/E2B 是 5V 信号。固件特意把编码器定时器 remap 到了**耐 5V 的 PA15/PB3/PB4/PB5**，所以可以直连、不用电平转换。**前提是按表接对**：E1A→PA15、E1B→PB3、E2A→PB4、E2B→PB5；**千万别接到旧的默认脚 PA0/PA1/PA6/PA7（那四个不耐 5V，会烧）**——那四个脚现在是空的。

2. **HC-SR04 的 ECHO 接 PA12（5V 耐受脚），可直连、不用分压。** PA12 是 FT，固件配成浮空输入，5V 回波直接接没问题。注意 **PA12 是 USB D+**，大多数蓝色药丸在它上面有个 ~1.5k 上拉到 3.3V——对 HC-SR04 的推挽回波无影响（它能拉高拉低）。我们不接 USB，所以 PA12 空着可用。想要余量可选加 1k 串，但不必。

3. **OLED 用 PB6/PB7 GPIO 当电源有限流隐患。** STM32 单脚约 20mA（绝对最大 25mA），0.96″ SSD1306 满屏点亮（电荷泵）峰值 10–20mA，**正卡在单脚极限**，可能略欠压/满白偏暗。**更稳：VCC 接 MD220A 3V3、GND 接公共地**，PB6/PB7 留空；屏显内容稀疏也能压电流。

4. **MPU6050 用 3.3V 供电。** GY-521 标 3.3V/5V 都行（板载 LDO），但**板载上拉和 INT 的参考是 VCC**。VCC 喂 5V，INT 就朝 5V 漂——SCL/SDA 在 FT 的 PB10/PB11 还好，**但 INT 在非 FT 的 PA5，会被打坏**。所以 **MPU VCC 接 MD220A 3V3**。

5. **不要给 ~6V 的 MC130 电机上 12V！** MD220A 的**输入电压 = 电机 VM**，固件满 PWM 会全开。输入 12V ≈ 2 倍过压 → 过转、过热、烧电机/齿轮箱。**给 MD220A 输入 ~6–7.4V**（2S 锂电常用）；实在要用高压，就在固件里把 `MOTOR_PWM_MAX` 降下来让平均电压 ≈6V，并重调增益。（MD220A/TB6612 本身耐到 ~15V，烧的是电机不是板。）

6. **必须共地（单点星形地）。** VM / 5V / 3.3V 都从 MD220A 出来，**但每个模块的 GND（电池负极、MD220A GND、C8T6 GND、各模块 GND）必须汇到同一点**。少接一根地，编码器/回波读乱码、IMU 乱跳。电机回流（PWM 高 di/dt）走粗地，别和传感器/逻辑地串一条细线。

7. **BOOT0 跳线接 GND 才能正常从 Flash 运行**（BOOT0=0 → 从 0x08000000 启动）；只有用串口 bootloader 烧录时才把 BOOT0 拉高。**SWD（PA13/PA14）保留**给下载/调试，固件不复用。

> 额外提醒：非 FT 的脚（PA0/PA1/PA2/PA3/PA5/PA6/PA7、PB0、PC13）都别误飞到 5V 轨；其中 PA0/PA1/PA6/PA7 现在是空脚（编码器已挪走），更要注意别顺手接 5V。上电前用表逐一量 VCC 和各信号静态电平。

## 上电与首次调试（BRING-UP 清单）

按顺序做，**先架空轮子**：

- [ ] **量电源**：上电先用万用表确认 MD220A 的 5V≈5V、3V3≈3.3V，公共地连通。
- [ ] **核对编码器脚**：E1A/E1B/E2A/E2B 确实接到 PA15/PB3/PB4/PB5（不是 PA0/PA1/PA6/PA7）。
- [ ] **横放测倒地保护**：把车横放（大角度），看是否触发 **FallProtect 断电**（电机被切力）。
- [ ] **扶正轻推自启动**：扶到接近直立、轻推一下，应**自动起平衡**。
- [ ] **确认 IMU 绕 Y 轴（pitch）**：前后倾要落在 pitch 轴；装歪了改 MPU 轴向/正负号。
- [ ] **电机方向若反就翻 AIN/BIN**：车自己往一边冲说明电机接反——对调该电机 AIN1/AIN2（或 BIN1/BIN2），或翻固件方向位。
- [ ] **编码器方向**：速度反馈方向反了就对调那一对 A/B；左右反了对调 E1↔E2。
- [ ] **HC-05 配对 + 遥控**：手机蓝牙搜到 HC-05 配对（PIN 1234/0000），用 Serial Bluetooth Terminal 发 `F/B/L/R/S` 试行驶（USART2=9600）。
- [ ] **避障**：发 `O` 打开（OLED 显示 `AV:ON`）；手在车前 <25cm 晃一下，应 停→原地扫描→转向较空方向。再发 `O` 关闭。
- [ ] **OLED 显示**：实时显示 **避障状态 / ARM(或 FALL) / 角度 / Vx·Vw / 距离**；不亮先查 PB6–PB9 和供电（见 §注意3）。

> 第一次上电务必架空轮子；指令为 0 时电机应不动，倾斜后转向正确，再落地。
